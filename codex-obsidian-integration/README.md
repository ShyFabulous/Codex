# Codex 与 Obsidian 联动

Codex 适合处理代码、整理信息与自动化执行；Obsidian 则适合沉淀知识、连接想法与长期回顾。把两者结合，可以让笔记不止被保存，也持续参与实际工作。

## 可以怎样配合

- 在 Obsidian 中维护项目背景、需求、决策记录和待办事项。
- 让 Codex 读取相关笔记，基于已有上下文编写代码、修改文档或梳理方案。
- 将 Codex 产出的实现说明、复盘和变更摘要写回 Obsidian，形成可检索的项目知识库。

## 一个简单工作流

1. 在 Obsidian 新建项目笔记，记录目标、约束和参考资料。
2. 将笔记内容或文件位置交给 Codex，请它完成具体任务。
3. 将完成结果、关键取舍和后续行动补充回项目笔记。

这样，Obsidian 负责保存脉络，Codex 负责推进执行；每次协作都会让下一次工作更快进入状态。
